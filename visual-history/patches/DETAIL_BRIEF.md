## 1. 反幻觉硬约束（违反就重做）

### 1.1 WebSearch 何时强制
**必须 WebSearch** 的场景（不允许凭记忆/训练数据写）：
1. **具体年份** —— 写 "前 219 年""1842 年" 这种精确数字时
2. **数字** —— 兵力、年龄、人数、距离等具体数字
3. **引文/诗句** —— 任何被打引号的文字必须 WebSearch 核对原文
4. **关键事件因果链** —— 谁杀了谁、谁救了谁的具体顺序
5. **较冷门的人物** —— 你训练数据可能记错的（如周亚夫细狱细节）

**可以不 WebSearch**（训练数据可靠）的场景：
- 大常识：秦始皇是嬴政、唐太宗是李世民
- 朝代起讫年份的"约略"表述（如"汉武帝在位约半世纪"，但精确数字仍要查）

### 1.2 不确定时怎么办（按优先级）
1. **先 WebSearch 多搜几次**——可能多源都说同一件事，那就可信
2. **多源冲突时**：写最保守、最少争议的版本，加"据说""相传""一说"
3. **完全不确定**：**直接跳过该细节**，不要凑字数

### 1.3 绝对禁区
- ❌ 编造未发生的事件
- ❌ 把演义/小说情节当作历史（如"草船借箭"是《三国演义》虚构，正史只有"草船受箭"零散记载——细微差别要照实写）
- ❌ 编造具体的对话/独白
- ❌ 把模糊年代写成精确日期
- ❌ 把可能性写成必然性（如"可能是""疑为"的事件不要写成"是"）

## 2. 长度分等级（务必遵守）

按知名度三档，长度上下限：

| 等级 | 标准 | 长度（中文字符）|
|---|---|---|
| **A 大人物/重大事件** | 中小学历史课本主条目（秦始皇、汉武帝、唐太宗、武则天、忽必烈、朱元璋、康熙、孙中山等；安史之乱、靖康之耻、鸦片战争、辛亥革命等） | **400–600** |
| **B 中等知名** | 中学历史课本有提的人物/事件（商鞅、张良、岳飞、王安石、于谦、张居正等；澶渊之盟、玄武门之变、土木堡之变等） | **250–400** |
| **C 较冷门** | 偶尔出现但相对小众（如 杨素、王彦章、檀道济、贾鲁等；高平之战、采石矶之战等） | **150–250** |

**段落分隔**：用一个空行（`\n\n`）分段。建议 A 级 3–4 段、B 级 2–3 段、C 级 1–2 段。

**风格**：故事化、有节奏感，不是百科条目流水账。可参考第一轮 STYLE_BRIEF.md §1 风格守则。

## 3. 写作要求

### 3.1 必备元素（A/B 级）
- **关键时间节点**（年份要核对）
- **关键事件/转折**（不只是出生—做官—死亡的流水）
- **经典故事细节**（如商鞅立木为信、唐太宗与魏徵的对话样态、岳飞十二道金牌等耳熟能详的桥段，必须出现）
- **后世影响/反讽/命运感**（保持项目一贯的"耐人寻味"调性）

### 3.2 互链
- 用 `[[id]]` 互链其他人物/事件/政权（凡是有 id 的实体都尽量链）
- 在 detail 里互链密度可比 bio 高 2–3 倍

### 3.3 不要做的
- 不要重复 bio 已有的内容（detail 是补充，不是扩写）
- 不要写"综上""总而言之"这种总结句
- 不要现代政治判断
- 不要诗词/语录长段引用（>20 字的引用要谨慎，必须 WebSearch 核对原文且只引最有名的一两句）

## 4. 工作流（每个 entity）

1. **读现有 bio/narrative** —— 知道什么已经写了，避免重复
2. **判断等级**（A/B/C），决定长度
3. **WebSearch 1–3 次**（按需）—— 查关键年份、引文、人名、因果
4. **写 detail**（200–600 字按等级）
5. **自检**：所有具体数字/引文/人名都有 WebSearch 依据吗？

## 5. patches/<你的 id>.json 格式

```json
{
  "agentGroup": "D3",
  "scope": "qin / chuhan / western-han / eastern-han 的所有 keyPersonIds + keyEventIds",
  "peopleDetails": [
    {
      "id": "qinshihuang",
      "level": "A",
      "detail": "嬴政生于赵国邯郸，十三岁继位秦王，二十二岁亲政。在位前期由 [[qin-lvbuwei]] 摄政，后借平定 [[qin-laoai]] 之乱顺势削吕、亲掌大权。\n\n前 230 年起十年内灭韩、赵、魏、楚、燕、齐六国，前 221 年完成统一。自创『皇帝』之号（取『三皇五帝』各一字），废分封行郡县，统一文字（小篆）、度量衡、车轨，修长城、驰道，南征百越、北却匈奴。\n\n但严刑峻法、徭役繁重，又行 [[fenshu-kengru]]，民怨蓄积。前 210 年第五次东巡途中暴卒于沙丘平台，年五十。[[lisi]] 与 [[qin-zhaogao]] 在 [[qin-shaqiu-zhimo]] 中合谋篡改诏书，赐死长子 [[qin-fusu]]，立胡亥为帝——他亲手锻造的帝国，开始向悬崖滑落。\n\n他的两面性是中国史最早的标本：开创了延续两千年的中央集权骨架，又用最暴烈的方式让自己的王朝十五年而亡。"
    }
  ],
  "eventDetails": [
    {
      "id": "qin-tongyi",
      "level": "A",
      "detail": "前 230 年至前 221 年，秦王嬴政以『远交近攻』为总方略，依次灭韩（前 230，内史腾）、赵（前 228，[[wj-wangjian]] 破邯郸，前 222 灭代）、魏（前 225，王贲水灌大梁）、楚（前 223，王翦六十万破项燕，前 222 灭楚南境）、燕（前 222，王贲俘燕王喜）、齐（前 221，王贲不战而下临淄）。十年间，分裂五百余年的诸夏归一……"
    }
  ]
}
```

### 字段说明
- `agentGroup`：你的 agent id（D1–D8）
- `scope`：一句话说你覆盖了哪些 era 的什么实体
- `peopleDetails[]` 和 `eventDetails[]` 是两个数组
- 每条只需 `id` + `level` + `detail` 三个字段（`level` 用于汇总统计）
- 已有 entity 没 `detail` 字段的就新建；不要去改 bio/narrative
